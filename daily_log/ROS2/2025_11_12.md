### 4. 문제점 및 해결 과정 (Issues & Solutions)
P1. 24시간의 장시간 훈련에도 불구하고 validation ACCURACY가 오히려 2.148% 감소(기존 35.579% -> 33.431%)하는 현상 발생. 전이학습으로 ImageNet에서 학습된 가중치를 모두 가져온 yolov8n 모델의 50%보다 낮다는 점에서 합성 데이터와 실제 데이터간 차이가 극명하게 갈린다는 점을 확인. 이번 sim2real 작업에 어울리지 않을 수 있다는 판단.
-> S1. 사전 계획한 데이터 생성전략을 유지한채 이미지 수를 기존 1,600장에서 3,200장으로 확대, 기존 yolov8n 모델 적용, 데이터 증강 수준 약 15% 향상(mosaic 0.3 -> 0.5, translate 0.0 -> 0.1 적용), 옵티마이저 변경(AdamW -> SGD)

P2. 평소 메모리 수준은 23%였으나 Antigravity 에디터만 실행하면 약 85%까지 치솟는 현상 발생. 직접적인 문제 원인은 훈련을 위해 열어둔 폴더가, 정확히 타겟으로 지정된 폴더가 아닌 상위 폴더를 열어두었기 때문임을 확인
-> S2. Traffic Monitor를 활용해 PC 리소스가 할당되는 부분을 하나하나 확인하여 낭비되는 리소스를 모아서 보다 개선된 모델과 학습 환경 구현(현재 훈련 중)

P3. Windows 환경에서의 C++ 런타임 라이브러리 의존성 문제: `TrafficMonitor` 로컬 배포 중 `VCRUNTIME140.dll` 및 `MSVCP140.dll` 누락 오류 발생. (개발 2)
-> S3. Visual C++ Redistributable 설치 및 의존성 해결: 해당 오류가 C++ 런타임 환경 부재로 인한 것임을 파악하고, 최신 Visual C++ Redistributable 패키지를 설치하여 의존성 문제를 신속히 해결.

P4. TCP/IP 소켓 통신 보안 취약점 (제3자 접속): 1:1 채팅 시스템 구현 중 인가되지 않은 제3자의 접속 시도 시 예외 처리 부족으로 인한 보안 우려 및 연결 오류 가능성. (개발 3)
-> S4. 접속 차단 및 예외 처리 로직 구현: 허용된 IP 외의 접속 요청 발생 시 `ConnectionRefusedError`를 발생시키거나 강제로 연결을 종료하는 화이트리스트 기반 차단 로직을 구현하여 통신 보안 강화.

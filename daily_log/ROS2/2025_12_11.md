### 4. 문제점 및 해결 과정 (Issues & Solutions)
P1. TCP 소켓 통신의 비동기 입력 처리 문제: 단일 스레드 기반의 소켓 프로그램에서 특정 문자(스페이스 등)의 경우 클라이언트의 연속적인 입력 스트림을 처리하지 못하고 메인 루프가 블로킹(Blocking)되는 문제 발생. (개발 1)

-> S1. 병렬 처리(Multi-threading) 모델의 필요성 확인: 해당 문제의 근본적인 원인이 단일 프로세스의 순차적 처리에 있음을 확인하고, 논블로킹(Non-Blocking) I/O 또는 별도의 Thread를 활용한 병렬 처리 모델 도입을 다음 단계 계획으로 수립.

P2. AI 보조 도구 사용 시 컨텍스트 유실 및 지시사항 망각 현상: AI(Jemini Pro 3)를 활용한 장기 대화에서 초기 지시사항(Persona)이 희미해지는 컨텍스트 유실(Context Drift) 문제 발생. (개발 2)	

-> S2. '스킵 커넥션' 기반 프롬프트 관리 전략 수립: AI의 답변 품질 유지를 위해, 초기 지시사항을 반복하여 삽입하는 AI Skip Connection 유사 전략을 적용하여 컨텍스트 일관성을 성공적으로 복구 및 유지. (AI 활용 메타인지 증명)


P3. 저사양 환경에서의 GPU 메모리 자원 비효율성 문제: Kaggle YOLO 모델 학습 중 메모리 한계와 흩뿌려진(Fragmented) GPU 메모리 할당으로 인해 학습 시간이 과도하게 길어지는 성능 저하 문제 발생. (개발 3)	

-> S3. 정적 메모리 진단 및 학습 환경 최적화: 로컬 PC의 GPU 메모리 할당 상태를 직접 진단하고, 한정된 자원 내에서 배치 크기(Batch Size) 조정 및 메모리 단편화 문제를 관리하는 학습 환경을 구현하여 학습 지속성을 확보.

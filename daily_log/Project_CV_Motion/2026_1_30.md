# [2026-01-30] 프로젝트 일일 보고서 (Daily Report)

## 1. 문제 인식 (Problem Identification)
* **통합 시스템의 구조적 이해 필요:** CV(객체인식), TTS/STT(음성 상호작용), ROS2(제어)가 결합된 복잡한 시스템의 데이터 흐름(Data Flow) 파악이 최우선 과제로 식별됨.
* **환경 및 목표 정의:** 단순 기능 구현을 넘어 '주방'이라는 특정 공간 내에서 로봇과 인간이 공존하기 위한 위치 정의(Workspace Context) 및 시나리오 구체화 필요.

## 2. 접근 방식 (Approach)
* **Base-code 역설계:** 제공된 베이스코드의 노드(Node) 간 종속성과 토픽(Topic) 통신 구조를 분석하여 시스템의 확장 가능성 확인.
* **전략적 역할 분배 (R&R):** 팀원의 강점을 고려하여 모듈별 담당자 지정 및 인터페이스 정의.
    - 로봇 제어 (HG, WJH): Manipulator 모션 및 좌표계 관리.
    - 객체 인식 (RJH, SH): YOLO 모델 기반 도구 식별 및 시선 트래킹 연구.
    - 일정 관리 및 가이드 (CH): PM 역할 및 전체 개발 프로세스 표준화.
* **시나리오 기반 설계:** '사용자 인식 - 메뉴 선정 - 조리 보조 - 서빙'으로 이어지는 사용자 경험 중심의 흐름도 작성.

## 3. 해결 및 구현 내용 (Implementation)
* **베이스코드 분석 및 실습:**
    - TTS/CV/ROS2 통합 노드 구동 확인.
    - 각 노드별 발행/구독(Pub/Sub) 토픽 모니터링을 통해 제어 주기 및 데이터 병목 지점 파악.
* **프로젝트 환경 구성:**
    - Position 1 (사용자), Position 2 (조리도구), Position 3 (조리대)로 구분된 좌표계(Frame) 설계.
    - 5종 조리도구(집게, 뒤집개, 냄비, 칼, 양념통)에 대한 작업 우선순위 선정.
* **협업 체계 가동:** 팀 회의(8회) 및 전문가 세션(5회)을 통해 도출된 피드백을 기반으로 '사용자 시선 트래킹' 등 고도화 기능 추가 확정.

## 4. 기반 기술 (Tech Stack)
* **Middleware:** ROS2 (Robot Operating System)
* **AI & Vision:** YOLO (Object Detection), User Eye Tracking
* **Interaction:** STT/TTS (OpenAI API/Custom Module)
* **Task Management:** Team Meeting (8 Sessions), Mentoring Session (5 Sessions)

## 5. 트러블 슈팅 (Troubleshooting)
* **이슈: 외부 API(TTS/STT) 인증 및 통신 지연**
    - **원인:** 베이스코드 실행 중 API Key 인식 오류 및 네트워크 레이턴시로 인해 로봇 제어 노드로의 데이터 전달이 간헐적으로 끊기는 현상 발생.
    - **해결:** 환경 변수(Environment Variable) 설정을 통한 인증 방식 표준화 및 비동기(Asynchronous) 호출 로직 검토를 통해 제어 루프의 실시간성 확보 시도.
    - **학습:** 단순 코드 실행보다 '통신 환경의 안정성'이 시스템 전체 퍼포먼스에 미치는 영향을 데이터로 확인.

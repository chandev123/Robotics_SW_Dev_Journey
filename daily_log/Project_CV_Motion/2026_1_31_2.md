**일일 업무 보고**

**1. 문제인식**
* robot_control 노드에서 topic, service, action 통신을 주고받는 시스템 구축 필요
* robot_control.py에서 topic과 service 생성 및 기능 검토 필요
* 문제상황 정의:
  1) 문법 에러
  2) 런타임 에러
  3) 논리 에러
  4) 동시성 이슈 (deadlock, race condition)

**2. 접근방식**
* [일정] 주말 개발 목표 수립
  - 1.31(토): topic 구현
  - 2.1(일): service, action 구현 및 데이터 주고받는 노드 생성
* [제어] ROS2, CV 활용 매니퓰레이터 제어
* [구현] MultithreadedExecutor를 활용하여 기능 구현
* [구현] Client, Server, Callbackgroup 직접 작성
* [대응] Topic, Service 기본교재 재학습 및 Zero-base에서 빌드 재시작

**3. 해결**
* robot_control.py에서 'alive' topic 메시지 publish
* Zero-base 빌드 성공
* 동시에 여러 topic 발행 성공

**4. 기반기술**
* 하드웨어: m0609 로봇팔, RGB-Depth 카메라, PC
* 소프트웨어: ROS2, CV, robot_control.py
* 이론적 배경:
  - 프로그래밍 언어론, 컴파일러 이론 (문법/런타임 에러 대응)
  - 소프트웨어 공학, 소프트웨어 테스팅, V-model (논리 에러 대응)
  - 운영체제론, 병행 컴퓨팅 (동시성 이슈 대응)

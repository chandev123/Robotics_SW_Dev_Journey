## 1. 통합 실행 프로그램과 음성인식 간 ROS 2 통신 구현

### 🚩 문제인식 (Problem Definition)
기존 베이스 코드와 신규 구현 프로그램을 결합하여 실제 하드웨어를 제어하는 통합 시스템 구성이 필요함.

### 🎯 접근방식 (Approach)
* 통합 실행 프로그램(Runner)과 음성인식 프로그램(Kitchen Voice) 간의 ROS 2 통신 구조 설계.
* 비동기 명령 처리를 위해 ROS 2 Action 통신 방식 채택.

### 💡 해결 (Solution)
* 시스템의 핵심 엔진 역할을 하는 Runner 프로그램에 ROS 2 통신 인터페이스(Action Server/Client) 기능을 추가하여 통합 완료.

### 🛠 기반기술 (Tech Stack)
* ROS 2 (Action Interface)
* Python

### 🐛 트러블 슈팅 (Troubleshooting)
* **증상:** 정상 작동하던 베이스 코드를 이식했으나, 실행 시 과거의 로그가 출력되거나 오작동 발생.
* **분석:** 컨테이너, 코드 로직, 배포 구성을 점검했으나 이상 없음. 빌드 아티팩트 충돌로 판단.
* **조치:** 워크스페이스 초기화(`rm -rf build install log`) 및 재빌드 수행 후 정상 작동 확인.
